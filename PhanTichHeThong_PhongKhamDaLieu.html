Luồng Hoạt Động Tổng Thể Của Hệ Thống

Admin tạo tài khoản cho nhân viên → Nhân viên đăng nhập → Phân quyền theo vai trò → Thực hiện nghiệp vụ → Kết thúc quy trình.

🛡️ ADMIN – TẠO TÀI KHOẢN NHÂN VIÊN — frmQuanLyNhanVien / frmTaoTaiKhoan
Bước khởi đầu bắt buộc. Chỉ Admin mới có quyền tạo tài khoản đăng nhập cho nhân viên mới (bác sĩ, lễ tân). Nhân viên không thể tự đăng ký — mọi tài khoản đều được Admin cấp và phân quyền. Luồng này đảm bảo kiểm soát tập trung về bảo mật nhân sự.
Điều kiện tiên quyết
  • Chỉ Admin (MaVaiTro = 1) mới truy cập được form này
  • Admin phải đăng nhập trước bằng tài khoản gốc (seed data)
  • Tài khoản Admin gốc được tạo thủ công qua script SQL khi triển khai
Thông tin tạo tài khoản
  • HoTen, SoDienThoai, Email — Thông tin cá nhân
  • TenDangNhap — Phải UNIQUE, kiểm tra trùng ngay khi nhập
  • MatKhau — Admin đặt mật khẩu tạm, nhân viên đổi lần đầu đăng nhập
  • MaVaiTro — Admin chọn: Bác sĩ (2) hoặc Lễ tân (3)
Logic & Ràng buộc DB
  • UNIQUE TenDangNhap — DB tự chặn trùng tên đăng nhập
  • UNIQUE SoDienThoai — Mỗi nhân viên 1 SĐT riêng
  • MatKhau lưu dạng hash (SHA256/BCrypt), không lưu plaintext
  • INSERT vào bảng NhanVien với FK → VaiTro
  • Admin có thể khóa (vô hiệu hóa) hoặc đặt lại mật khẩu bất kỳ lúc nào
Patch SQL cần thêm
  • Thêm cột TrangThaiTK BIT DEFAULT 1 vào NhanVien (1=Hoạt động, 0=Khóa)
  • Thêm cột DoiMatKhau BIT DEFAULT 1 — Bắt đổi MK lần đầu
  • Kiểm tra TrangThaiTK = 1 trong logic đăng nhập
1
🔐 ĐĂNG NHẬP HỆ THỐNG — frmDangNhap
Nhân viên nhập TenDangNhap + MatKhau do Admin đã cấp. Hệ thống truy vấn bảng NhanVien → JOIN với VaiTro → kiểm tra TrangThaiTK = 1 (không bị khóa). Nếu là lần đầu đăng nhập (DoiMatKhau = 1) → bắt buộc đổi mật khẩu trước khi vào hệ thống. Dựa vào MaVaiTro, chuyển hướng sang Dashboard tương ứng.
Logic xác thực đầy đủ
  • SELECT NhanVien WHERE TenDangNhap = @user
  • Kiểm tra TrangThaiTK = 1 (chưa bị khóa)
  • So sánh MatKhau (hash BCrypt)
  • Nếu DoiMatKhau = 1 → frmDoiMatKhau
  • Lấy MaVaiTro → phân quyền màn hình
3 Nhánh phân quyền
  • 1 - Admin: Toàn quyền hệ thống
  • 2 - Bác sĩ: Khám, kê đơn, hồ sơ
  • 3 - Lễ tân: Tiếp nhận, lịch hẹn
Bảo mật
  • Sai mật khẩu → đếm số lần sai
  • Sai ≥ 5 lần → SET TrangThaiTK = 0 (khóa)
  • Chỉ Admin mới mở khóa lại được
  • Lưu phiên đăng nhập (Singleton)

👩‍💼 LỄ TÂN – TIẾP NHẬN BỆNH NHÂN
Lễ tân kiểm tra bệnh nhân trong hệ thống bằng SĐT. Nếu chưa có → tạo mới bảng BenhNhan. Sau đó gọi SP để tạo đồng thời Phiếu Khám + Dịch vụ cơ bản + Hóa đơn chờ thanh toán trong 1 Transaction. Phiếu khám được tạo với TrangThai = 0 (Chờ khám).

🔴 SQL Patch Cần Bổ Sung Vào NhanVien (Hỗ trợ luồng đăng ký / đăng nhập)
Thêm cột vào bảng NhanVien:
ALTER TABLE NhanVien
ADD TrangThaiTK BIT DEFAULT 1,
    DoiMatKhau BIT DEFAULT 1;
-- TrangThaiTK: 1=Hoạt động, 0=Khóa
-- DoiMatKhau: 1=Bắt buộc đổi MK lần đầu
Thêm cột TrangThai vào PhieuKham + MaLichHen FK:
ALTER TABLE PhieuKham
ADD TrangThai TINYINT DEFAULT 0,
    MaLichHen INT NULL,
    FOREIGN KEY (MaLichHen)
    REFERENCES LichHen(MaLichHen);
-- 0:Chờ, 1:Đang khám, 2:Xong, 3:Đã TT
Tạo bảng HinhAnhBenhLy (lưu ảnh bệnh lý):
CREATE TABLE HinhAnhBenhLy (
  MaHinhAnh INT IDENTITY(1,1) PK,
  MaPhieuKham INT NOT NULL FK,
  DuongDanAnh NVARCHAR(500) NOT NULL,
  GhiChu NVARCHAR(200),
  NgayChup DATETIME DEFAULT GETDATE()
);
